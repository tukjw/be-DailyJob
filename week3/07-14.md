### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단: 댓글 갯수를 표시하는 기능을 구현할때 게시글 테이블에 댓글 갯수 집계 컬럼을 만들지않았다
- 그렇게 판단한 이유: 게시글 목록을 보는 기능이 없고, 게시글 본문을 하나씩 보는 구조에 댓글을 게시글과 같이 불러와야해서 불러온 댓글 갯수를 직접 세는게 더 효율적이라고 느꼈다.
- 고려했지만 선택하지 않은 대안: 게시글 테이블에 댓글 갯수 집계 컬럼 만들어서 댓글 추가마다 갯수 관리하기
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향: 요구사항은 충족하며 집계 컬럼을 관리하는 로직이 없기때문에 코드의 복잡함이 줄어들어 유지보수에 더 좋을 것이다.

### 3. 가장 막혔던 문제와 해결 과정
- 문제 상황 또는 증상: H2 데이터베이스로 바꾼 후 부터 최신글이 계속 이상한 값으로 갱신 되었다
- 처음 추측한 원인: 영속성을 위해 H2 데이터베이스를 파일로 관리하였는데, 웹 서버에서도 H2 DB파일을 사용하며 동시에 H2 웹콘솔로 DB파일을 같이 사용해서 일관성이 깨지는 것 같았다.
- 원인을 확인하기 위해 시도한 방법: H2 웹 콘솔을 종료하여 파일을 동시에 사용하지 않도록 하였는데도 문제가 생겨 AI에게 질문하였다.
- 실제 원인: 테스트할때마다 H2 데이터베이스에 테스트용 데이터가 계속 쌓이던 것이었다
- 해결 방법: 테스트는 인메모리 H2로 하기로 하였다.
- 다음에 비슷한 문제가 생기면 먼저 확인할 것: 테스트가 실제 데이터가 쌓이는 구조인지 확인해야겠다고 생각했다
