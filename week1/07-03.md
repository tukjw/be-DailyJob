- 처음 추측한 원인: content-type이 image/avif 일땐 .svg가 브라우저에서 안보이는것 같았다.
- 원인을 확인하기 위해 시도한 방법: 크롬 개발자모드를 통해 .svg 확장자가 어떤 content-type으로 응답되는지 확인해보았다.
- 실제 원인: 지원하지 않는 확장자에대해서는 Accept헤더의 첫번째 value를 content-type으로 응답했는데, image/avif가 첫번째 value였고 브라우저에서 svg를 avif이미지형식으로 읽으려고하니 읽히지가 않았다.

### 1. 내가 최종 선택한 가장 잘 작성했다고 생각하는 프롬프트
원본 프롬프트: 자바 gradle 프로젝트를 기반으로 실행할때 실행되는 경로를 알려줘

선택한 이유: gradle 프로젝트 기반임을 명시한점이 좋았던 것 같다.

### 2. 원본 프롬프트 구조 분석
- 문제 상황: static 디렉토리의 경로를 찾을 수 없었다.
- 원하는 결과를 어떻게 요구했는가: 자바를 실행할때 실행되는 경로를 알려달라고 하였다.
- 필요한 맥락이 포함되어 있는가: 현재 static 디렉토리를 찾는다는 맥락은 포함되어있지 않았다.
- 판단 기준이나 제약 조건이 있는가: 없었다.
- 출력 방식이 명확한가: 명확하지않았다.
- 나의 가설, 우려, 의도가 드러나는가: 왜 실행되는 경로를 찾는지 이유를 명시하지않아 가설, 우려, 의도가 드러나지 않았다.

```

프로젝트 구조는 대략 다음과 같습니다.

├─ build.gradle

* IntelliJ에서 실행할 때 기준 경로가 달라질 수 있는 이유
* `File`로 접근하는 방식과 `ClassLoader.getResource()`로 접근하는 방식의 차이
* static 리소스를 읽는 추천 방법
```
